Introduction 简介

This document targets technical developers of merchant systems, who should have a background in web programming, understanding of RESTful API, and familiarity with HTTP, HTTPS request and response formats and processes, as well as JSON format.

本文档适用于商户系统的技术开发人员，要求具有WEB 程序开发背景，了解RESTful API, HTTP、HTTPS 请求和应答的格式和过程，对JSON 格式有所掌握。

Authentication 认证

This API uses Bearer Token for authentication. Please make sure to include the bearer token in the Authorization header for all requests.

Obtaining Bearer Token

To obtain the bearer token, you need to follow these steps:

  1. Go to our ClientLogin method. 通过 ClientLogin 登陆
  2. Sign in with your credentials using our MD5 method as follows. 登陆方式如下，签名，验签时用到的key 值均为商户密钥
// 如登录接口：C#语法 
string key=”3fg49gh7ye5yfjwhw67efj”;//密钥 secretKey provided by us

// Employing a descending secretKey.
// 我们将采用降序的secretKey
string descKey = new string (key.ToLower().OrderByDescending(c => c).ToArray()); 

// Parameters of the query except “Sign” should pass in ascending order, example as below 
// 除了“Sign”参数之外，查询的参数应按升序传递，示例如下：
// E.g. [I]jk[L][M]
string sign = MD5(“isMobile=0&loginId=cs003&merchantId=test&key=”+ descKey);//签名 sign
  1. Upon successful authentication, you will receive the bearer token.

Using Bearer Token

Once you have obtained the bearer token, include it in the Authorization header of your requests as follows:

Authorization: Bearer <your-access-token>
